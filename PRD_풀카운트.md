# PRD (Product Requirements Document)

## 문서 정보
- **프로젝트명**: 풀카운트 (Full Count) - 야구 팬 커뮤니티 및 직관/티켓 양도 플랫폼
- **작성자**: 안진경
- **작성일**: 2026-03-27
- **버전**: v1.0

---

## 1. 프로젝트 개요

### 1.1 프로젝트 목적

```
야구 팬들에게 팀별 소속감을 부여하는 전용 커뮤니티를 제공하고,
사기 피해 걱정 없는 안전한 에스크로 기반 티켓 양도 및 직관 동행 매칭 시스템을 구축하여
팬들의 야구 관람 경험을 개선하고 건전한 팬 문화를 조성한다.
```

### 1.2 프로젝트 범위

**포함 범위:**
- [x] 회원 관리 (가입, 응원 팀 선택, 뱃지 등급 관리)
- [x] 커뮤니티 관리 (팀 전용 비밀 게시판, 통합 게시판)
- [x] 직관/양도 관리 (동행 모집, 티켓 양도글 작성, 에스크로 결제)
- [x] 실시간 소통 (1:1 채팅 및 직관 그룹 채팅)

**제외 범위:**
- [ ] KBO 공식 티켓 예매 시스템 연동 및 발권 (개인 간 양도만 취급)
- [ ] 야구 외 타 종목 커뮤니티
- [ ] KBO 실시간 중계 영상 스트리밍

### 1.3 주요 이해관계자 (Stakeholders)
| 구분 | 역할 | 주요 관심사 |
|-----|------|------------|
| **야구 팬 (일반)** | 시스템 사용자 | 같은 팀 팬들과의 소통, 직관 메이트 구하기 |
| **티켓 양도/양수자** | 시스템 사용자 | 사기 위험 없는 안전한 티켓 거래 |
| **시스템 관리자** | 시스템 운영자 | 암표 및 비매너 유저 모니터링, 플랫폼 안정성 |
| **운영팀 (CS)** | 고객 지원 | 거래 분쟁 조정, 신고 접수 및 처리 |

## 2. 비즈니스 도메인 분석

### 2.1 핵심 비즈니스 프로세스

#### 2.1.1 회원 가입 및 팀 인증 프로세스

**상세 플로우:**
1. **회원가입 단계**
   - 기본 정보 입력 (이메일, 닉네임, 비밀번호)
   - 응원하는 야구팀 선택 (1개 팀 한정)
   - 이메일 인증 완료 후 가입 처리
2. **활동 및 등급 단계**
   - 가입 즉시 해당 팀의 '기본 뱃지' 부여
   - 직관 인증 및 커뮤니티 활동량에 따라 뱃지 등급 자동 업그레이드
   - 선택한 팀의 전용 게시판 접근 권한 획득

#### 2.1.2 티켓 양도 및 에스크로 결제 프로세스

**상세 플로우:**
1. 양도자가 티켓 정보(구역, 좌석 번호 일부 가림, 가격)와 인증샷을 포함하여 양도글 등록
2. 양수자가 게시글 확인 후 '양도 요청' 클릭
3. 1:1 채팅방 자동 개설 및 상세 조율
4. 조율 완료 시 양수자가 시스템에 대금 결제 (에스크로 상태로 보관)
5. 양도자가 실물 티켓 전달 또는 모바일 티켓 선물하기 완료
6. 양수자가 '인수 확정' 클릭 시 양도자에게 대금 정산 완료

**비즈니스 규칙:**
- 양도 가격은 정가를 초과할 수 없음 (프리미엄 제한)
- 결제 완료 전까지는 다른 사용자도 양도 요청 가능 (선결제 우선)
- 인수 확정 지연 시, 경기 시작 24시간 후 자동 확정 처리

### 2.2 비즈니스 이벤트
| 이벤트 | 트리거 | 결과 |
|--------|--------|------|
| **가입 및 팀 선택** | 이메일 인증 완료 | 회원 생성, 팀 뱃지 부여, 전용 권한 획득 |
| **양도 요청** | 양수자 버튼 클릭 | 게시글 상태 변경(예약중), 채팅방 개설 |
| **대금 안전결제** | 양수자 결제 완료 | 결제 대금 에스크로 보관, 양도자에게 알림 |
| **인수 확정** | 양수자 확정 클릭 | 대금 정산, 거래 상태(완료) 변경 |
| **비정상 거래 신고** | 유저 신고 접수 | 해당 게시글 노출 차단, 관리자 알림 |

## 5. 도메인 객체 (Domain Objects)

### 5.1 도메인 객체 식별 매트릭스
| 도메인 객체 | 유형 | 중요도 | 비고 |
|------------|------|--------|------|
| **Member (회원)** | Entity | 높음 | 핵심 사용자 |
| **Team (팀)** | Entity | 중간 | KBO 구단 정보 |
| **Post (게시글)** | Entity | 높음 | 커뮤니티 및 양도글 |
| **Transfer (양도/거래)** | Entity | 높음 | 에스크로 결제 비즈니스 |
| **ChatRoom (채팅방)** | Entity | 중간 | 유저 간 소통 |

### 5.2 상세 도메인 객체 정의

#### 5.2.1 Member (회원)
**역할**: 플랫폼을 이용하며 팀을 응원하고 티켓을 거래하는 사용자

**주요 속성:**
- `id`: 고유 식별자
- `email`: 로그인 이메일
- `nickname`: 커뮤니티 활동명
- `team`: 응원 팀 (외래키)
- `badgeLevel`: 뱃지 등급 (ROOKIE, PRO, ALL_STAR, LEGEND)
- `mannerTemperature`: 매너 온도 (거래 및 동행 평가 지수, 기본 36.5)

**주요 행동 (메서드):**
- `changeTeam()`: 응원 팀 변경 (시즌당 1회 제한)
- `upgradeBadge()`: 활동 점수에 따른 뱃지 승급
- `updateMannerTemperature()`: 리뷰 후 매너 온도 갱신

**비즈니스 규칙:**
- 닉네임은 중복될 수 없음
- 응원 팀이 설정되어야만 커뮤니티 글 작성이 가능
- 매너 온도가 일정 수치 이하로 떨어지면 거래 및 동행 모집 제한

#### 5.2.2 Post (게시글)
**역할**: 커뮤니티 소통 및 동행/양도 모집을 위한 글

**주요 속성:**
- `id`: 고유 식별자
- `author`: 작성자 (외래키)
- `boardType`: 게시판 종류 (TEAM_ONLY, GENERAL, MEETUP, TRANSFER)
- `title`: 제목
- `content`: 내용
- `matchDate`: 경기 날짜 (MEETUP, TRANSFER인 경우 필수)
- `status`: 상태 (OPEN, RESERVED, CLOSED)

**주요 행동 (메서드):**
- `reserve()`: 양도/동행 예약 중으로 상태 변경
- `close()`: 모집 완료 및 마감 처리
- `isEditable()`: 수정 가능 여부 확인 (예약 중엔 수정 불가)

#### 5.2.3 Transfer (양도 거래)
**역할**: 양도자와 양수자 간의 안전한 티켓 거래 상태 관리

**주요 속성:**
- `id`: 고유 식별자
- `post`: 연관된 게시글 (외래키)
- `seller`: 양도자 (외래키)
- `buyer`: 양수자 (외래키)
- `price`: 거래 금액
- `status`: 거래 상태 (REQUESTED, PAYMENT_COMPLETED, TICKET_SENT, COMPLETED, CANCELLED)

**주요 행동 (메서드):**
- `payEscrow()`: 양수자 결제 완료 처리
- `confirmTransfer()`: 양수자 인수 확정 및 정산 지시
- `cancelTransfer()`: 거래 취소 및 환불 처리

**비즈니스 규칙:**
- 거래 금액은 0원(무료 나눔)이거나 게시글에 명시된 금액과 일치해야 함
- 양수자가 결제한 금액은 `COMPLETED` 상태가 되기 전까지 플랫폼이 보관
- 결제 완료 후 취소는 양도자와 양수자의 상호 동의가 필요

#### 5.2.4 ChatRoom (채팅방)
**역할**: 동행 조율 및 티켓 양도를 위한 실시간 대화 공간

**주요 속성:**
- `id`: 고유 식별자
- `post`: 연관된 게시글 (외래키)
- `roomType`: 채팅방 타입 (ONE_ON_ONE, GROUP)
- `createdAt`: 생성일

**주요 행동 (메서드):**
- `joinMember()`: 채팅방 인원 추가
- `exitMember()`: 채팅방 퇴장

## 6. 도메인 관계도

### 6.1 개념적 관계도

```
[Team]     1 ──── N [Member]
[Member]   1 ──── N [Post]
[Post]     1 ──── 1 [Transfer]
[Post]     1 ──── 1 [ChatRoom] (1:1 거래용)
[Post]     1 ──── 1 [ChatRoom] (1:N 동행 그룹)
```

### 6.2 관계 상세 설명
| 관계 | 카디널리티 | 설명 | 제약조건 |
|------|------------|------|----------|
| **Team ↔ Member** | 1:N | 한 구단은 여러 팬(회원)을 가짐 | 회원은 1개 구단만 선택 |
| **Member ↔ Post** | 1:N | 한 회원은 여러 게시글을 작성할 수 있음 | 정지된 회원은 작성 불가 |
| **Post ↔ Transfer** | 1:1 | 하나의 양도 게시글에서 하나의 거래가 성사됨 | 거래 게시판(TRANSFER)에 한함 |
| **Post ↔ ChatRoom** | 1:1 | 양도글 하나당 조율을 위한 1:1 채팅방 생성 | 동행 모집글은 1:N 그룹채팅 |
